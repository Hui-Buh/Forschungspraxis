README:


Der Ordner Bericht enthält alle Dateien, die ich im Bericht verwendet habe.
Ein gesammeltes Projekt ist als .zip hinterlegt, ein .pdf enthält den fertigen Bericht.

/Images ist der Referenz Ordner, der in der Gui angegeben wird, um festzulegen, welche Bilder in den Berechnung verwendet werden sollen.
Unterordner werden nicht miteinbezogen. 

/Daten Ordner enthalten die Patienten Daten. Jeder Patient hat einen eigenen Ordner mit seiner ID als Namen.

==> Es werden bei den meisten Funktionen nur Daten verarbeitet, bei denen ein Bild und das passende Datenpaket vorliegt.
==> Alle Skripte arbeiten mit .mat Dateien. (.sacc Dateien in unserem Format können in der Gui umgewandelt werden.)

/Matlab Code enthält den gesammelten Code zur Umsetzung der Funktionalitäten. 

/Amsterdam beinhaltet den Code, den ich für Jonas in Amsterdam geschrieben habe.

/Figures beinhaltet alle Bilder, die ich mit meinem Programm schon mal erzeugt habe.
Mit dem aktuellen Code erstellte Bilder sind in die dortigen Unterordner eingepflegt.

/my_gbvs enhält den modifizierten code zur saliency map Berechnung auf der Basis der von Jonathan Harel bereitgestellten Matlab Skripte.
http://www.vision.caltech.edu/~harel/share/gbvs.php

==> Saccade_Analyzer_gui.m verwaltet das user front end. Dies kann genutzt werden, um alle erstellten Funktinalitäten anzusteuern. 

==> Alle nicht genannten Ordner enthalten entwerder Backups oder für die Funktionalität unwichtige Dateien.
